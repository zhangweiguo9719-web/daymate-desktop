# DayMate 隐私说明

## 记录什么

开启活动记录时，DayMate 记录前台应用进程名称、可执行文件路径、开始与结束时间、活跃与空闲秒数，以及鼠标点击和键盘按键的次数汇总。可执行文件路径只用于从本机程序文件提取应用图标，不读取该路径下的其他文件。只有用户主动开启“窗口标题记录”时才记录窗口标题。

DayMate 使用 Windows 提供的最近键鼠输入时间判断电脑是否真的在使用。连续 5 分钟没有输入后，保持亮屏的时间记为空闲，不计入活跃时间。

## 不记录什么

DayMate 不记录具体按键、输入内容、鼠标坐标或点击目标，不截屏，不读取聊天、网页或文档正文，不录制麦克风或摄像头，不扫描程序图标以外的本地文件内容。

## 数据在哪里

桌面活动和可执行文件路径保存在操作系统为 `com.daymate.desktop` 分配的应用数据目录中的 `daymate.sqlite3`。提取后的图标只在应用运行期间保存在内存缓存，不上传、不另建图标档案。设置页会展示实际数据路径。任务和偏好在 MVP 阶段保存在 WebView 本地存储。

## 是否联网

智能音乐会向 Audius 发送不含个人信息的通用场景词或用户主动输入的歌曲搜索词，并从 Audius 官方流媒体地址播放所选音频。任务标题、活动明细和窗口标题不会随音乐请求发送；网络不可用时自动使用安装包内的 CC0 曲目。

AI 为可选功能。服务商、Base URL 和模型名称保存在本机普通设置中；API Key 按服务商分别保存在 Windows 凭据管理器，不写入 SQLite、WebView localStorage、日志、Git 仓库或安装包。点击“测试连接”时只发送固定测试短句，不发送任务或活动数据。

用户主动点击“AI 按今日节奏推荐”时，只发送当前音乐偏好、今日活跃分钟数和未完成任务数量，不发送任务标题、窗口标题、文档内容或应用活动明细。

## 删除与关闭

用户可以关闭活动采集、保持窗口标题关闭，并在“数据与隐私”页清空活动数据。清空操作需要二次确认。

## 隐私问题

请通过 GitHub 的 `Privacy concern` Issue 模板报告。公开日志前请移除用户名、路径和窗口标题。
